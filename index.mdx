ytdl-archiver is a Python command-line tool that downloads YouTube playlists using yt-dlp and produces media-server-ready sidecar files alongside each video. Whether you're building a personal Jellyfin library or preserving a channel for long-term archiving, ytdl-archiver handles format selection, metadata generation, subtitle embedding, and thumbnail downloads automatically.

## What ytdl-archiver produces

Every successful archive run creates a structured set of files alongside each downloaded video:

- **Media file** — `.mp4` (or configured format), selected at maximum quality
- **NFO sidecar** — Kodi/Jellyfin/Emby-compatible XML with episode metadata
- **Metadata sidecars** — `.info.json` (yt-dlp) and `.metadata.json` (full run context)
- **Subtitles** — Language-tagged `.srt` files with optional embedding
- **Thumbnail** — `.jpg` sidecar for media server artwork

## Get started

<Steps>
  <Step title="Install ytdl-archiver">
    Install with uv (recommended) or pip. FFmpeg must be on your `PATH`.

    ```bash
    uv tool install ytdl-archiver
    ```
  </Step>
  <Step title="Run first-time setup">
    The interactive setup wizard generates your config files automatically.

    ```bash
    ytdl-archiver init
    ```
  </Step>
  <Step title="Define your playlists">
    Edit `~/.config/ytdl-archiver/playlists.toml` to add the channels or playlists you want to archive.
  </Step>
  <Step title="Archive">
    Run the archive command to start downloading and generating sidecars.

    ```bash
    ytdl-archiver archive
    ```
  </Step>
</Steps>
