---
title: "Quickstart: archive your first playlist"
description: "Go from a fresh install to a running archive in four steps: run init, define playlists.toml, then start your first ytdl-archiver archive run."
---

This guide walks you through the minimum steps to get ytdl-archiver running and archive your first YouTube playlist. By the end you will have config files in `~/.config/ytdl-archiver/` and a completed archive run producing media files and sidecar metadata.

<Steps>
  <Step title="Install ytdl-archiver">
    If you have not installed ytdl-archiver yet, the fastest path is `uv tool install`. FFmpeg must be on your `PATH` before you continue.

    ```bash
    uv tool install ytdl-archiver
    ```

    For full installation options — including pip, from-source, and FFmpeg setup — see the [Installation guide](/installation).
  </Step>

  <Step title="Run first-time setup">
    Run the interactive setup wizard to generate your configuration files:

    ```bash
    ytdl-archiver init
    ```

    The wizard creates two template files:

    - `~/.config/ytdl-archiver/config.toml` — global settings for download behavior, cookies, and output
    - `~/.config/ytdl-archiver/playlists.toml` — the list of playlists and channels to archive

    <Note>
      You do not need to run `init` explicitly. If `config.toml` is missing when you run any non-help command (such as `ytdl-archiver archive`), setup runs automatically and exits after completing. You can then edit the generated files and re-run your command.
    </Note>
  </Step>

  <Step title="Define your playlists">
    Open `~/.config/ytdl-archiver/playlists.toml` in your editor and add the channels or playlists you want to archive. Each entry uses a `[[playlists]]` block:

    ```toml
    [[playlists]]
    id = "UUxxxxxxxxxxxxxxxxxxxxxx"
    path = "Music/Example Channel"
    name = "Example Music Channel"

    [playlists.download]
    format = "bestaudio"
    write_subtitles = false
    write_thumbnail = true
    ```

    Key fields:

    | Field | Description |
    |---|---|
    | `id` | YouTube channel or playlist ID (the `UC…` or `PL…` identifier from the URL) |
    | `path` | Relative path under your archive base directory where files are saved |
    | `name` | Human-readable name used in NFO metadata and logs |
    | `[playlists.download]` | Optional per-playlist overrides for format, subtitles, thumbnails, and more |

    <Tip>
      You can use the `ytdl-archiver search` command to discover channel and playlist IDs interactively and append them to `playlists.toml` automatically.
    </Tip>

    <Note>
      Both `playlists.toml` and the legacy `playlists.json` format are supported. If both exist in the config directory, TOML takes precedence.
    </Note>
  </Step>

  <Step title="Run the archive">
    Start the archive with:

    ```bash
    ytdl-archiver archive
    ```

    ytdl-archiver will iterate through every playlist in `playlists.toml`, download new videos, and write sidecar files alongside each one. Videos already present in the playlist's `.archive.txt` file are skipped automatically.

    For each downloaded video you will get:

    - A media file (`.mp4` by default, remuxed away from WebM)
    - `<stem>.info.json` — yt-dlp native metadata sidecar
    - `<stem>.metadata.json` — ytdl-archiver full run context
    - `<stem>.nfo` — Kodi/Jellyfin/Emby-compatible episode XML (when enabled)
    - `<stem>.en.srt` (or other language codes) — subtitle files with optional embedding
    - `<stem>.jpg` — thumbnail sidecar (when `write_thumbnail = true`)
  </Step>
</Steps>

## Where config files live

| File | Purpose |
|---|---|
| `~/.config/ytdl-archiver/config.toml` | Global settings: archive directory, download defaults, cookies, output format |
| `~/.config/ytdl-archiver/playlists.toml` | List of playlists and per-playlist overrides |
