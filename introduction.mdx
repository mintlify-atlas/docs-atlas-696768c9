ytdl-archiver is a modern Python command-line tool that downloads YouTube playlists using yt-dlp and produces media-server-ready sidecar files alongside each video. It is designed for building structured archives compatible with Jellyfin, Emby, and Kodi — handling format selection, NFO generation, subtitle embedding, and thumbnail downloads automatically.

## What ytdl-archiver produces

Every successful archive run creates a structured set of files alongside each downloaded video:

- **Media file** — `.mp4` (or configured format), selected at maximum quality, remuxed away from WebM by default
- **NFO sidecar** — Kodi/Jellyfin/Emby-compatible XML with episode metadata, ratings, and genres
- **Metadata sidecars** — `.info.json` (yt-dlp native sidecar) and `.metadata.json` (full run context owned by ytdl-archiver)
- **Subtitles** — Language-tagged `.srt` files (for example `<stem>.en.srt`) with optional embedding
- **Thumbnail** — `.jpg` sidecar for media server artwork, when enabled

## Use cases

ytdl-archiver is a good fit if you want to:

- Build a Jellyfin, Emby, or Kodi library from YouTube channels or playlists
- Create a personal long-term archive with full metadata and subtitle records
- Automate recurring downloads with per-playlist format and quality preferences
- Backfill NFO and sidecar metadata for an existing yt-dlp archive

## Dependencies

Before installing, make sure these are available on your system:

| Dependency | Required | Notes |
|---|---|---|
| Python 3.14+ | Yes | Required at runtime |
| [`uv`](https://docs.astral.sh/uv/) | Yes (recommended install method) | Used to install and run ytdl-archiver |
| FFmpeg | Yes | Must be on your `PATH`; used for remuxing and subtitle embedding |
| Deno or Node.js | Recommended | Improves yt-dlp extraction compatibility |
| Firefox | Recommended | Enables browser cookie extraction |
| Rust | Optional | Only needed for setup TUI development |

<Note>
  If FFmpeg is not on your `PATH`, archive runs will fail at the remux step. See [Installation](/installation) for platform-specific instructions.
</Note>
